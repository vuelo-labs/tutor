Glossary Terms

Token

The unit of measurement AI models use to process text. Roughly 3–4 characters, or about ¾ of a word. Every prompt you send and every response you receive costs tokens. Most models have a limit on how many tokens they can process in one conversation — this is called the context window. Tokens also determine cost: the more you send, the more you pay on paid plans.

Unit of AI text processing. ~¾ word. Tokens in = cost + context consumed. Controls pricing and context limits.

Prompt

What you type and send to an AI model. A good prompt gives the model enough context to understand what you need, who you are, and what a good response looks like. A bad prompt is vague, assumes too much, or buries the actual request in unnecessary preamble.

Your input to the model. Contains: role + task + format + constraints. Quality of output is bounded by quality of prompt.

Model

The AI itself — the thing doing the thinking. Different models (Claude Sonnet, GPT-4o, Gemini 2.5 Pro) have different strengths, speeds, and costs. Picking the right model for the right task is part of getting good results without overspending.

The AI system generating responses. Each model has different capability, latency, and cost tradeoffs. Match model to task complexity.

Context Window

Think of it as the AI's working memory for a conversation. Everything in the window — your messages, its responses, any files you've shared — is what it can "see" and reason about. When a conversation runs long, older content can fall out of the window and the model effectively forgets it. This is why starting fresh threads and using persistent instructions matters.

Max tokens the model can attend to in one session. When exceeded, earlier content is dropped. Manage via: shorter threads, persistent instructions, re-seeding.

Persistent Instructions

Instructions you set once that apply to every conversation — without you having to repeat them. Think of it as the AI reading your onboarding doc before every chat. Where these live depends on your platform: Claude calls them Custom Instructions or Project instructions, ChatGPT calls them Custom Instructions, Gemini uses Gems, Claude Code uses CLAUDE.md. Exercise 1 is entirely about this.

Instructions loaded at session start, applied globally. Platform equivalents: Claude → Custom Instructions / Project instructions · ChatGPT → Custom Instructions · Gemini → Gems · Claude Code → CLAUDE.md.

System Prompt

A hidden block of instructions that runs before the conversation starts — usually set by the developer or platform, not visible to you as a user. When you set Custom Instructions on ChatGPT or Project instructions on Claude, they get compiled into the system prompt behind the scenes. You don't usually write these directly unless you're building your own AI tool.

Pre-conversation instruction block. Set by developer/platform. Custom Instructions, Gems, and CLAUDE.md all compile into the effective system prompt.

Thread

A single conversation with an AI model. Each thread has its own context window — it starts fresh and builds up as you exchange messages. Long threads degrade in quality as the window fills up or the model loses track of earlier context. Knowing when to start a new thread (and how to carry important context forward) is a core skill.

Single conversation session. Has isolated context window. Quality degrades over long threads. Start fresh + re-seed when needed.

Hallucination

When an AI confidently states something that is wrong or made up — a date that doesn't exist, a citation for a paper that was never written, a function that doesn't exist in a library. It sounds plausible because the model is very good at generating fluent, confident text. Hallucinations are more likely when asking about very recent events, niche topics, or when the model is guessing at specifics.

Model output that is confident but factually incorrect. Higher risk: recent events, niche topics, specific citations. Mitigate: ask for sources, verify outputs, use retrieval-augmented tools.

Reasoning Model

A type of AI model that takes extra time to think through a problem before responding — working through intermediate steps internally before giving you an answer. This makes them slower and more expensive, but much better at logic, maths, strategy, and complex multi-step problems. Examples: Claude with extended thinking, OpenAI's o3, Google's Gemini 2.5 Pro with thinking enabled.

Model that performs internal chain-of-thought before responding. Higher latency + cost. Better at: logic, maths, planning, multi-step tasks. Examples: o3, Claude extended thinking, Gemini 2.5 Pro thinking.

MCP (Model Context Protocol)

A standard, open protocol that lets AI models connect to external tools and data sources — your calendar, codebase, browser, databases — so they can act on the world rather than just respond with text. Think of it as a universal plug socket: any AI that supports MCP can use any tool built for MCP, without custom integration work. Claude invented it; most major platforms now support it.

Open protocol for connecting AI models to external tools + data. Enables action, not just generation. Supported by Claude, ChatGPT, Gemini, Tines. Claude invented it.

Temperature

A setting that controls how creative or predictable the model's responses are. Low temperature (near 0) means the model picks the most likely next word at every step — useful for factual, consistent, or structured output. High temperature means more randomness and creativity — useful for brainstorming or creative writing. Most chat interfaces hide this setting, but it's useful to know it exists.

Controls output randomness. Low (→0): deterministic, consistent, factual. High (→1+): creative, varied. Usually abstracted in chat UIs. Relevant when using the API directly.

Re-seeding

The practice of starting a new conversation thread and pasting in a summary of the context from a previous one. Useful when a thread has gone stale, hit the context limit, or wandered off track. Instead of trying to salvage a broken conversation, you start fresh and give the model exactly what it needs to know — no more, no less.

Starting a new thread with a distilled context summary from a previous one. Use when: thread is stale, context limit hit, or quality degraded. Carry signal, drop noise.
